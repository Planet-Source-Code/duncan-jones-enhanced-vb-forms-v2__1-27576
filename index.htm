  2001.08.09 Updated

The latest version of the EventCTL OCX Beta release 1.0.3.  This update has a new set of properties which allow you to control the way your form is resized. Setting the MaxTop, MaxWidth, MaxLeft and MaxHeight sets the size which your form will be when maximized. Setting the MaxTrackWidth, MinTrackWidth, MaxTrackHeight and MinTrackHeight controls the minimum and maximum sizes that the user can drag resize your form to.

To use the VBEventWindow control, add the following code to the form on which the control is placed:

Private Sub Form_Initialize()

  VBEventWindow1.ParentForm = Me.hWnd

End Sub

See the help documentation for more information

  2001.08.06 Updated

The latest version of the EventVB.dll Beta release.  This release fixes some defects and extends the object model with the introduction with the top level ApiSystem class which is used to manipulate information that pertains to the entire computer.

The EventVB.dll is an ambitious ActiveX dll which provides a Document Object Model type of access metaphor to the Windows Operating System by way of wrapping the complexities of the API in a programmer friendly object oriented package.  The control is in a very early Beta stage and as such the documentation is fairly rudimentary. 

We will post examples of using the control as they are developed.  If you would like us to tackle and particular area of the API that you are having difficulty with, please contact the development team. 

 

  2001.07.21 New 

The Image Map OCX mimics the activity of the HTML Client Side Imagemap in a Visual basic OCX.  The control can also read well formatted HTML for compatability with your existing internet applications.

  2001.07.20 Updated 

The Global Hotkey Control is a control that you simply drop onto your Visual Basic form, set the properties and you have a system wide hotkey that will trigger whether or not your application is the foreground application. The event can be used for anything, not just activation as per some hotkey examples.

All of these downloads are dependent on the Visual Basic 5 runtime environment which is not included with them.  If you are using a development environment other than Visual Basic 5 you will need to download the runtimes from Microsoft.  For Visual Basic 6 users we suggest you recompile the source code to avoid having to ship two run times. 

Source Code

  2001.07.21 New 

The Image map source code shows how the image map control works, including some useful tips for creating an intuitive programmer interface for an OCX control and how to subclass the usercontrol to allow you to find out the coordinates at which mouse events occur.

The control uses the API calls: GetWindowLong, SetWindowLong, CallWindowProc and DefWindowProc to show you how to use subclassing from within Visual Basic, and shows how to split a single long value into its HiWord and LoWord components to track the coordinates at which mouse events occur.

  2001.07.20 Updated 

The Global Hotkey source code demonstrates how to register a hotkey, create new window messages and respond to them by subclassing the UserControl main window. It also demonstrates a clever technique for having more than one subclassed control on a form at a time.

It uses the API calls: RegisterWindowsMessage, RegisterHotkey and GlobalAddAtom and also shows the use of subclassing to intercept messages being sent to a control.
